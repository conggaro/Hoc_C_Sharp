Sau khi máy tính đã có IP của website, nó vẫn chưa vào được web ngay. Nó phải làm 2 việc quan trọng:

1️⃣ Tìm đường đi trên Internet → Routing
2️⃣ Thiết lập kết nối với server → TCP handshake

1️⃣ Routing – tìm đường đến server 🌐

Internet giống như mạng lưới rất nhiều router.

Máy tính của bạn sẽ:

Gửi gói tin tới router trong mạng nhà

Router này chuyển tiếp tới router của ISP

Các router trên Internet tiếp tục chuyển tiếp gói tin

Cuối cùng gói tin tới server của website

Sơ đồ đơn giản:


Máy tính
   │
   ▼
Router nhà
   │
   ▼
ISP router
   │
   ▼
Router Internet
   │
   ▼
Server website




Mỗi router sẽ nhìn vào địa chỉ IP đích rồi quyết định:

"Gửi gói tin này sang router nào để gần đích hơn?"

Cơ chế này gọi là Internet Protocol routing.

2️⃣ TCP Handshake – bắt tay kết nối 🤝

Sau khi gói tin tới đúng server, máy tính cần thiết lập kết nối đáng tin cậy với server bằng Transmission Control Protocol.

Quá trình này gọi là 3-way handshake.

Bước 1 – SYN

Máy tính nói:

Xin chào server, tôi muốn kết nối
Bước 2 – SYN-ACK

Server trả lời:

OK, tôi nhận được yêu cầu
Bước 3 – ACK

Máy tính xác nhận:

Tốt, bắt đầu trao đổi dữ liệu

Sơ đồ:

Client → SYN → Server
Client ← SYN-ACK ← Server
Client → ACK → Server

Sau bước này kết nối TCP được thiết lập.



3️⃣ Sau đó mới gửi yêu cầu web

Bây giờ trình duyệt mới gửi request:

GET / HTTP/1.1
Host: google.com

Server gửi lại:

HTML
CSS
JavaScript
Images

Trình duyệt dựng thành trang web.

Tóm tắt toàn bộ quá trình truy cập website
1. Nhập google.com
2. DNS → tìm IP
3. Routing → tìm đường đến server
4. TCP handshake → thiết lập kết nối
5. HTTP request → tải trang web

Nếu gói tin đi tới 192.168.1.x → gửi trong mạng nội bộ
